# Contributing

## Branches

- Use `main` as the primary branch.
- Small changes can be committed directly when appropriate.
- For larger changes, open a short-lived branch and merge it back into `main`.

## Pull Requests

- Keep PR titles clear and descriptive.
- Briefly explain what changed and how it was checked.
- Prefer focused PRs over large mixed updates.

## Merge Policy

- Use Pull Requests when collaboration or review is helpful.
- Do not force-push or rewrite shared history unless the team agrees.
- Keep the repository simple: no extra branch workflow rules are required.
